# 设置页精致化与响应式统一设计

- 日期: 2026-06-17
- 状态: 已批准，等待实施计划
- 目标: 将设置页改造成宽度统一、层次清晰、响应式可靠、视觉精致的专业工具设置界面

## 背景

当前设置页已经拆分为页面壳、二级导航、设置原语和多个 section，但视觉体验仍不稳定：

- 设置页内层布局缺少稳定的全宽约束，内容区在部分窗口下没有自然吃满剩余空间。
- 不同 section 的卡片、提示、AI 配置面板和模型表格存在各自的视觉节奏，页面不够好记。
- 控件右侧宽度没有统一规则，输入框、按钮、滑块和开关在窄屏下容易显得散。
- AI 设置页是复杂度最高的区域，线路卡片、推荐横幅、配置表单、模型列表没有统一成一套设置页语言。
- 关于页存在更偏营销化的装饰视觉，和开发工具设置页的安静工作感不一致。

## 设计方向

采用“精致工具台设置页”方案：安静、克制、可扫描，优先服务长期使用和快速定位。

| 维度 | 决策 |
|---|---|
| 页面结构 | 左侧二级导航 + 右侧统一内容面板 |
| 内容宽度 | 内容区居中，统一最大宽度 960px，始终可自适应压缩 |
| 视觉基调 | 浅灰工作区、白色分组、8px 圆角、细边框、轻阴影 |
| 控件节奏 | 行式设置项左文右控，桌面对齐，窄屏换行 |
| AI 设置 | 收敛为同一套设置原语，不做卡片套卡片 |
| 响应式 | 桌面、平板、窄屏均无横向滚动，无文字/控件重叠 |

## 信息架构

保留现有设置分类，避免改变用户记忆路径：

1. 通用设置
2. 快捷键
3. 外观设置
4. 自动解码
5. AI 助手
6. 关于

左侧导航保持固定宽度。宽屏显示图标和文字；窄屏收缩为图标栏，并通过 tooltip 暴露完整名称。右侧内容区域负责滚动，外层应用布局不产生页面滚动。

## 页面布局

设置页根容器：

- `h-full w-full min-w-0 flex-1`
- 背景使用 `bg-default-50`，暗色下使用 `dark:bg-default-50/5`。
- 内层横向容器也必须 `w-full min-w-0`，确保内容区占满剩余宽度。

内容区：

- `flex-1 min-w-0 overflow-y-auto`
- padding 使用响应式节奏：窄屏 12px，普通屏 20px，宽屏 24px。
- 内容包裹层统一 `mx-auto w-full max-w-[960px]`。
- AI 设置可以复用同一宽度，不额外放宽，避免页面宽度在不同 tab 间跳动。

## 设置原语

现有 `settingPrimitives.tsx` 继续作为设置页视觉系统入口，但需要收敛规则。

### SectionHeader

- 标题保持紧凑，桌面不超过 24px。
- 描述使用 13-14px，作为页面意图说明，不承担操作说明。
- Header 和首个分组之间保持统一间距。

### GroupLabel

- 不使用大写字母样式，中文场景下保持自然可读。
- 左右 padding 与卡片内容对齐。
- 负责建立分组记忆点，文字短而明确。

### SectionCard

- 圆角统一为 8px。
- 边框使用 `border-default-200/70`。
- 阴影极轻，只用于从背景中抬起分组。
- 支持 `divided` 行分割。
- 避免页面中卡片套卡片；复杂内容用表单区块或分割线处理。

### SettingRow

- 左侧: 32px 图标块 + 标题 + 描述。
- 右侧: 控件容器使用固定/最小宽度规则，桌面对齐。
- 窄屏: 行可换行，右侧控件铺满或右对齐，但不溢出。
- 描述文本允许换行，不截断关键信息。

### ChoiceCard

- 圆角统一为 8px。
- 选中态使用细边框和轻背景，不使用强烈装饰。
- 卡片内部预览图形保持固定高度，避免选中态造成布局跳动。

### InfoNote

- 作为内嵌提示，不成为独立大卡片。
- 色彩只表达语义：primary 表示信息，warning 表示风险。
- 文本控制在短段落，避免设置页变成说明书。

## 各 Section 设计

### 通用设置

继续按“外观 / 编辑器 / 数据 / 危险操作”分组。深色模式、展开 Tab 栏、本地持久化使用统一 `SettingRow`。默认缩进大小也改为同一行式结构，右侧滑块宽度在桌面固定，窄屏可伸缩。

危险操作保留红色语义，但按钮不做过度强调。关闭本地存储和重置应用的提示应清楚表达影响。

### 快捷键

保持两行设置项。快捷键输入框和“按组合键设置”提示在桌面横向排列，在窄屏上下排列。说明块放在卡片底部，使用 `InfoNote`，不改变快捷键捕获逻辑。

### 外观设置

聊天样式和字体大小继续使用选择卡片，但卡片尺寸、圆角、选中态、预览区域统一。桌面两列或三列，窄屏单列。选择后保留现有 toast 反馈。

### 自动解码

保留四个解码器开关。所有开关行高度一致，图标色调用于快速识别。底部说明保持简短，强调全局生效和性能影响。

### AI 助手

AI 设置页使用同一套设置视觉，避免特殊化：

- AI 线路区域使用统一线路卡片，卡片圆角、边框、间距与其他选择卡片一致。
- 推荐 SSOOAI 提示改为 `InfoNote`，避免视觉重心过重。
- 配置面板使用 `SectionCard` 内的表单区块，不再出现 `rounded-2xl` 的内部卡片。
- API 地址、API 密钥、模型选择、测试连接按照表单顺序排列。
- 测试连接栏在桌面右对齐，窄屏控件全宽。
- 模型列表保留横向滚动兜底，但表格容器不能撑破页面。

### 关于

关于页从装饰型 hero 改为产品信息面板：

- Logo、名称、版本信息保持在一个简洁品牌区。
- 功能特点和技术支持使用同样的分组卡片。
- 减少大渐变、大光晕和“专业版”徽标带来的营销感。

## 数据流与行为

本次只重构 UI，不改变设置状态语义：

- `useSettingsStore` 字段和 setter 不变。
- `useOpenAIConfigStore` 字段、线路启用逻辑、模型增删逻辑不变。
- 主题切换、侧边栏展开、默认缩进、持久化确认、快捷键捕获、解码器副作用、AI 测试连接、模型增删保持原行为。
- toast 文案可微调，但不得删除关键反馈。

## 错误处理

- 关闭本地持久化仍必须弹出确认弹窗。
- AI 测试连接失败继续显示失败原因。
- uTools 线路不可用时继续清楚展示不可用状态。
- 模型列表为空时继续展示空状态。
- 输入框、表格、按钮在窄屏下不得超出容器。

## 验收标准

- 设置页所有 tab 内容宽度一致，切换 tab 不出现明显宽度跳动。
- 桌面宽度下内容居中，右侧控件对齐，分组层次清楚。
- 窄屏宽度下没有横向滚动，文字、按钮、输入框、表格不重叠。
- AI 设置页视觉与其他设置页统一，配置面板可读、可操作。
- 深色模式下边框、背景、文字对比正常。
- `pnpm build` 通过。
- 浏览器验证 `/settings`，至少覆盖桌面和窄屏视口；若 in-app Browser 策略再次拦截本地地址，需记录该阻塞并使用可用的本地验证证据补充。

## 非目标

- 不重写设置存储。
- 不新增设置项。
- 不更换 UI 框架。
- 不改变主应用侧边栏和编辑器页面。
- 不做营销落地页风格设计。
